🔍 PriceWatch 價格異常稽核報告

🔍 系統運作機制說明 (點擊展開/收起)

本系統透過底層運算引擎與歷史快照庫進行交叉比對，將超市定價的異常波動進行客觀分級與標示：

  • 🟣 促銷包裝更改 (差異大)：折實價維持不變，但促銷字眼被大幅度修改 (英文基準相似度 < 50%)。系統判定為改變促銷條件但無實質價格下調。
  • 🟢 促銷字眼微調 (差異小)：折實價不變，促銷字眼英文基準相似度達 50% 以上。通常為超市後台更正錯字或微調排版格式，提供 Audit Trail 供查核。
  • 🟠 無效促銷 (Invalid Promo)：超市為貨品掛上促銷標籤，但經系統精算後，發現單件折實價 大於或等於 上次歷史紀錄，實質未有產生折扣。
  • 📈 隱性加價 (Hidden Increase)：無論有沒有掛上特價牌，只要今日折實價 高於 上次歷史紀錄，即被判定為加價。
  • 📉 真降價 / 🥈 次平真降價：今日折實價 低於 上次歷史快照記錄之折實價，再結合全城對手定價決出最平霸主與次平後備選項。
  • ⚖️ 雙重數據透視 (Fairness Protocol)：系統會嚴格比對超市原價及原始折實價與「全城平均公價」的距離。若跨超市出現折實同價，系統將秉持公平原則，優先顯示「原價最貼近全城公價」的超市，杜絕發水標高。
